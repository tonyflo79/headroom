# Domain docs

This is a single-context repository.

## Before exploring

- Read `CONTEXT.md` at the repository root when it exists.
- Read ADRs under `docs/adr/` that touch the area being changed.
- If either location does not yet exist, proceed without treating its absence as an error.

## Vocabulary

Use the domain terms defined in `CONTEXT.md` in issue titles, PRDs, implementation plans, test names, and code. Do not drift to synonyms the glossary explicitly avoids.

If a needed concept is absent, first check whether existing project language already covers it. Record a genuine terminology gap for the domain documentation rather than silently inventing competing language.

## Architectural decisions

Surface any conflict with an existing ADR explicitly. Do not silently override a recorded decision; identify the ADR, explain the conflict, and state why reopening the decision may be warranted.
